## Project Overview

"Alice Smart Charging" — a self-contained animated SVG sequence built with GSAP 3.12.7 and vanilla JS. Five SVG compositions animate in sequence (enter → hold → exit → next), looping infinitely. Ships as two standalone HTML files: dark theme (`index.html`) and light theme (`index-light.html`). Designed for iframe embedding (e.g., Framer).

## How to View / Test

No build step. Open directly in a browser:

```bash
open SmartCharging/index.html
open SmartCharging/index-light.html
```

## Architecture

### File structure

- `SmartCharging - 0[1-5].svg` — Source SVG compositions (698×627). Scenes 01 and 02 contain large base64-embedded raster images (~290KB each).
- `gsap.min.js` — Local copy of GSAP 3.12.7 (avoids CDN dependency).
- `index.html` / `index-light.html` — Fully self-contained: all 5 SVGs and GSAP are inlined. ~725KB each.

### Build process

The HTML files are generated by a Python build script (run inline) that:
1. Reads all 5 SVG files and `gsap.min.js`
2. Inlines them into a single HTML with the animation JS
3. For light mode, prepends a `fixLightColors()` function that remaps colors per-scene

To rebuild after editing SVGs or animation code, re-run the Python build script from the conversation history.

### Animation system

Each scene is a function (`s1()` through `s5()`) returning a GSAP timeline. `buildMaster()` chains them with `repeat: -1`.

Element selection uses **index-based slicing** on `directChildren(sceneNum)` — the direct children of each SVG excluding `<defs>`. The exact element indices per scene are:

| Scene | Bars/Lines | Bubbles | Face | Labels | Other |
|-------|-----------|---------|------|--------|-------|
| 1 | ch[5-12] lines | — | — | — | ch[1] title, ch[2-4] ellipses, ch[13] bolt, ch[14] image |
| 2 | — | — | — | — | ch[1-2] arcs, ch[3-5] toggle, ch[6-8] search, ch[9] image |
| 3 | — | — | — | — | ch[2-5] card1, ch[6-10] card2, ch[11-15] card3, ch[16-17] close |
| 4 | — | — | — | ch[8-14] rows | ch[2-4] left card, ch[5-7] right card, ch[15-18] badges |
| 5 | ch[2-28] bars | ch[29-30] main, ch[31-34] chat | ch[37-47] | ch[48-55] | ch[35-36] indicator |

### Light mode color strategy

Global safe remap handles backgrounds and grays. **Does not** globally remap `white` or `#F5F5F5` — those require per-scene context (e.g., `white` is fox face in scene 5 but bell icon in scene 3). Per-scene fixes in `fixLightColors()` handle:
- Scene 3: bell icon `white` → `#334155` (visible on light cards)
- Scene 4: value text `#F5F5F5` → `#1E293B`
- Scene 5: fox face `white` stays white; "20" bubble → `#E8F0FE`; text labels with `#364157` → `#475569`

### Key constraints

- SVG elements have native `transform="rotate(...)"` attributes (scenes 3 card tilts). GSAP `rotation` property **overrides** these — do not animate `rotation` on elements that have SVG rotation transforms. Use `x`/`y`/`opacity` only for entry.
- Scene 2 toggle: knob travels exactly 44px right (OFF→ON). Label fades out simultaneously as knob slides over it.
- Scene 1 ellipses: enter from bottom (`y: 80` → `y: 0`), not from center scale.
